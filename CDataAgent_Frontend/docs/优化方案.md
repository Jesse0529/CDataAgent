# CData Agent 前端优化方案

## 1. 目标与范围

本方案围绕工程化、流式会话可靠性、首屏性能、代码简洁性和可维护性推进。第一阶段只修改前端项目；不改变后端接口路径、HTTP 方法、请求参数名称或既有 SSE 事件名称。

当前生产构建可以通过，但 `npm run lint` 有 21 项 error、3 项 warning。首个优化目标是让质量检查成为可执行的门禁，并消除流式会话在断线与取消时的状态错误。

## 2. 已确认的前后端 SSE 契约

前端使用：

- `POST /apis/agent/chat/stream?message=...&fileIds=...`
- `POST /apis/agent/chat/resume?conversationId=...&resumeToken=...&resumeSeq=...`

后端事件名称为 `message`（或无名称文本）、`status`、`chart`、`table`、`ping`、`complete`、`error`。`status` 会先下发 `resumeToken:<token>`；`complete` 为 JSON，包含 `type`、`chartOption`、`tokenUsage` 和 `resumeToken`。

第一阶段保持以上契约不变。

### 断点续传的边界

后端缓冲区维护了事件序号并按 `resumeSeq` 回放，但 SSE 输出没有把该序号写入 `id` 字段，也未在 data 中传给前端。因此，前端无法准确知道“最后成功接收的后端事件序号”。

本阶段会保证前端不会在重连尚未结束时提前把消息标记为完成，并能正确取消重连；但如果要达到严格的“零丢失、零重复”续传，需要后端后续将序号通过 SSE `id` 字段（推荐）或事件 payload 暴露给前端。这是一项兼容性增强，不能由前端单独完成。

## 3. 优先级与实施路线

### 第一阶段：可靠性与质量门禁

1. 将流请求的结果建模为完成、失败、用户取消三种明确状态；错误不再被吞掉后伪装为成功结束。
2. 将重连纳入同一个 await 生命周期；停止、页面卸载时取消当前请求、重连定时器和动画帧。
3. 清理流请求超时计时器，支持 CRLF/LF 两种 SSE 分隔符，并移除非空断言。
4. 修复 Biome 的格式、import 排序和规则错误；新增 `typecheck`、`test`、`check` 脚本。
5. 增加 SSE 解析、取消、超时和错误路径的单元测试。

验收标准：`npm run check`、`npm run build` 和测试命令均通过；主动停止后不再发起重连；连接失败后消息不会被误标记为完成。

### 第二阶段：性能与渲染

1. 让两个图表入口都异步加载图表弹窗；使用 `echarts/core` 按需注册图表、组件和 Canvas 渲染器。
2. 流式更新只修改当前消息，不在每一帧遍历全部消息；仅当用户停留在底部时自动滚动。
3. 长对话引入窗口化/分页渲染；本地缓存设置消息数和字节上限，必要时迁移到 IndexedDB。
4. 为 bundle 大小、首屏加载和长会话渲染建立可观测指标与预算。

#### 实施记录（2026-07-15）

- 已将配置面板中的图表预览改为异步组件，图表代码不再进入首次路由加载链路。
- 流式 token 更新改为直接更新当前消息，避免每一帧复制整条消息数组；用户离开底部浏览历史时暂停自动滚动。
- 本地会话缓存限制为最近 100 条、1.5 MB，超限时从最早消息开始淘汰。
- 保留完整 ECharts 运行时以兼容 AI 生成的任意图表类型；按需注册需要先将后端图表类型契约收紧后再实施。

### 第三阶段：模块化与长期维护

1. 从 `AnalysisPage.vue` 拆出 `useConversation`、`useFiles`、`useAgentStream`。
2. 从 `ChatMessage.vue` 拆出 Markdown/表格渲染与图表入口；统一保留一种安全的 Markdown 渲染策略。
3. 将重复的 GET/POST/DELETE/upload 逻辑收敛为统一 request client，支持超时、取消、错误归一化和可选重试。
4. 增加 `.env.example`、CI 工作流、组件测试和端到端的模拟 SSE 回归测试。

## 4. 风险控制

- 所有第一阶段改动保持后端 API 兼容，不改后端代码。
- 先用单元测试覆盖解析与状态迁移，再执行完整构建验证。
- 不自动重试上传和非幂等提交；重连只用于既有的 `/chat/resume` 接口。
- 若后端后续暴露 SSE `id`，前端再升级为精确序号恢复，并以新旧事件格式兼容方式灰度接入。
